# ASR Performance Specification

**Status**: targets defined; acceptance gated on Part E GPU benchmark run (not yet executed — skipped this dispatch, gates separately per operator sign-off).

**Scope**: defines the measurable performance targets for the hardened 8-stage ASR pipeline introduced in the 2026-04-22 ASR hardening dispatch (Track 3, Parts A-D, F, G).

---

## 1. Latency targets

| Stage | Target | Percentile | Hardware |
|---|---|---|---|
| RTF (full pipeline, VAD→Turn) | ≤ 0.2 | wall-clock / audio-seconds | GCP L4 (24 GB) |
| Per-segment transcription | ≤ 2.0 s | P95 | GPU |
| Claim extraction (Opus 4.7) | ≤ 3.0 s | P95 | any |
| End-to-end utterance-to-claim | ≤ 5.0 s | P50 | any |
| End-to-end utterance-to-claim | ≤ 7.0 s | P95 | any |

RTF target of ≤ 0.2 means a 10-second utterance should transcribe in ≤ 2 seconds wall-clock — consistent with published faster-whisper + CTranslate2 INT8 benchmarks on L4 class hardware (SYSTRAN benchmark large-v2 proxy, `research/asr_stack.md §2.1`).

## 2. Quality targets

| Metric | Target | Baseline (prior to hardening) |
|---|---|---|
| Overall WER | ≤ 10% | ~28% (Whisper baseline on medical, Arora arXiv 2502.11572) |
| Medical-term WER | ≤ 12% | ~28% (same source) |
| Hallucination rate | ≤ 0.5% of segments BLOCK-flagged on clean speech | not measured pre-hardening |
| Speaker diarisation DER | ≤ 10% (accuracy ≥ 90%) | not measured pre-hardening |
| Invented medications downstream | 0 (zero tolerance) | not measured pre-hardening |

Medical-term WER target of ≤ 12% is a hypothesis based on:
- `initial_prompt` medical vocabulary bias (documented effect: 3–8 pp reduction, `research/asr_stack.md §4.3`).
- LLM-based transcript cleanup (TranscriptCleaner) for major misspellings.
- Levenshtein word correction for residual single-char errors.

All numbers are **hypotheses until Part E GPU run lands.**

---

## 3. Degradation strategy (4-point ladder)

When latency or quality targets cannot be met, the following degradation ladder applies in priority order:

1. **Swap Whisper model**: large-v3 → distil-large-v3 (MIT, 6.3× faster RTF at some WER cost). Trigger: RTF > 0.4 on L4.
2. **Disable LLM cleanup**: `PipelineConfig(enable_cleanup=False)`. Cleanup adds ~200–500 ms per segment. Trigger: per-segment latency P95 > 3 s or `OPENAI_API_KEY` unavailable.
3. **Disable word correction**: `correct_medical_tokens` skipped. Levenshtein search over large vocabulary adds ~50–200 ms. Trigger: per-segment latency P95 still > 2.5 s after step 2.
4. **Disable audio preprocessing**: `PipelineConfig(enable_preprocessing=False)`. ffmpeg not available in all environments. Trigger: ffmpeg absent or normalisation adds > 500 ms for short clips.

Each step is observable via `telemetry.get_stats()` and the structlog event stream.

## 4. Telemetry instrumentation

The following pipeline stages are instrumented with `@telemetry.measure`:

| Stage name | Wraps |
|---|---|
| `normalize` | `preprocess.normalize_audio` |
| `trim` | `preprocess.trim_silence` |
| `transcribe` | `_Transcriber.transcribe` |
| `diarise` | `_AlignerDiarizer.align_and_diarise` |
| `cleanup` | `TranscriptCleaner.clean` |
| `guard` | `hallucination_guard.check` |
| `correct` | `word_correction.correct_medical_tokens` |

Aggregate statistics are available at any time via `telemetry.get_stats()`, which returns `{count, p50, p95, p99, max}` per stage.

## 5. Measurement methodology

Acceptance is gated on `eval/asr/run_asr_benchmark.py` (Part E, scheduled separately). The benchmark runs:
- Three ASR variants (A: baseline, B: biased+hardened, C: distil fallback) per `docs/asr_benchmark.md`.
- The same synthetic chest-pain clip (`eval/fixtures/asr/chest_pain_demo.wav`, generated by `synth_audio.py`).
- On GCP L4 24 GB spot (primary) or local 4090/A100 if available.
- RTF, overall WER, medical-term WER, and per-stage telemetry are recorded in `docs/asr_benchmark.md §4`.

**Numbers marked `TBM` (to be measured) in `docs/asr_benchmark.md` are not updated by this spec.** Only the Part E benchmark run populates those cells.
