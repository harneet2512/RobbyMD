# SYNTHETIC_DATA.md — Dataset Manifest

Every dataset, audio file, transcript, vignette, and labelled case used by this repository is listed below with its provenance. Per `rules.md §2.2`, nothing else enters the repo. `tests/privacy/test_no_phi.py` scans for real-looking identifiers and fails the build if any appear.

## Our synthetic content

None yet. As we author synthetic cases:

- Cases generated by an LLM: the generation prompt is checked in alongside the case.
- Cases recorded from actors reading a scripted dialogue: the script is checked in, plus an attestation in this file stating that no PHI appears.
- Every committed case carries a first-line sentinel:
  ```
  # SYNTHETIC — not real patient data. See SYNTHETIC_DATA.md.
  ```

## Rule recap (from `rules.md`)

- **§2.1** No real patient audio, transcripts, EHR records, clinical notes, or de-identified real data. No exceptions.
- **§2.2** Every dataset declared here with its provenance.
- **§2.3** No third-party clinical APIs implying a BAA. Anthropic Opus 4.7 is covered by Anthropic's standard terms.
- **§2.4** PHI sentinel test (`tests/privacy/test_no_phi.py`) must stay green.
- **§2.5** Nothing shown in the demo video could be mistaken for real patient data. Names are obviously fictional. Dates of birth are obviously synthetic.
