## Our synthetic content

### ASR demo clip — `chest_pain_demo.wav`

- **Declared**: 2026-04-21, `wt-extraction` Phase 1 ASR benchmark (`docs/asr_benchmark.md`).
- **Source**: generated by `src/extraction/asr/synth_audio.py::synthesise_clip` at build time. **No real-human audio is recorded.** The script is 10 scripted utterances authored from scratch during the hackathon window (rules.md §1.1); see `DEMO_SCRIPT` in that module for the full text.
- **Synthesis**: `pyttsx3` (MIT; SAPI5 / NSSpeechSynthesizer / eSpeak backend) alternating between two system TTS voices for physician / patient. No neural voice cloning. No real-person voice reference.
- **PHI attestation**: the script contains no real names, DOBs, MRNs, SSNs, addresses, or any patient-identifiable content. All characters are generic ("Good morning", "I take metoprolol and aspirin", etc.). The `tests/privacy/test_no_phi.py` sentinel covers the script module as part of its `src/` scan.
- **First-line sentinel** (in-file): `synth_audio.py` carries a module docstring declaring synthetic-only; the WAV itself is not committed.
- **Location**: regenerated per run into `eval/fixtures/asr/chest_pain_demo.wav` (not committed). Reproduction steps in `docs/asr_benchmark.md` §3.

As further synthetic cases are authored:

- Cases generated by an LLM: the generation prompt is checked in alongside the case.
- Cases recorded from actors reading a scripted dialogue: the script is checked in, plus an attestation in this file stating that no PHI appears.
- Every committed case carries a first-line sentinel:
  ```
  # SYNTHETIC — not real patient data. See SYNTHETIC_DATA.md.
  ```

## Rule recap (from `rules.md`)

- **§2.1** No real patient audio, transcripts, EHR records, clinical notes, or de-identified real data. No exceptions.
- **§2.2** Every dataset declared here with its provenance.
- **§2.3** No third-party clinical APIs implying a BAA. Anthropic Opus 4.7 is covered by Anthropic's standard terms.
- **§2.4** PHI sentinel test (`tests/privacy/test_no_phi.py`) must stay green.
- **§2.5** Nothing shown in the demo video could be mistaken for real patient data. Names are obviously fictional. Dates of birth are obviously synthetic.
